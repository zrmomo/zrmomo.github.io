retroft 的注解是什么时期解析

​ Retrofit 的注解是在运行时解析的。当创建一个 Retrofit 实例并通过它创建 API 接口的动态代理对象时，Retrofit 会解析接口中定义的所有注解。这包括请求方法的注解（如 @GET、@POST 等），参数的注解（如 @Path、@Query 等），以及其他用于配置请求和响应处理的注解。

Retrofit 动态代理解析

​ 理论上动态代理是为了通过自动生成代理对象在运行时控制要代理的对象的访问，JDK动态代理主要涉及两个类：java.lang.reflect.Proxy 和 java.lang.reflect.InvocationHandler，举个例子：

java.lang.reflect.Proxy 和 java.lang.reflect.InvocationHandler 是两个核心类，它们的作用如下：

  1. java.lang.reflect.Proxy

Proxy 类是动态代理类的超类，它提供了用于创建动态代理类和实例的静态方法。主要方法包括：

  • newProxyInstance(ClassLoader loader, Class<?>[] interfaces, InvocationHandler h): 这是创建动态代理实例最常用的方法。它需要三个参数：类加载器（用于加载代理类）、接口数组（代理类需要实现的接口列表）、调用处理器（用于处理方法调用的对象）。这个方法返回一个实现了指定接口的代理实例。
  1. java.lang.reflect.InvocationHandler

InvocationHandler 是一个接口，它定义了一个方法 invoke(Object proxy, Method method, Object[] args)。在使用动态代理时，你需要实现这个接口，并提供 invoke 方法的实现，这个方法会在代理实例上的方法被调用时执行。

  • invoke(Object proxy, Method method, Object[] args): 这个方法有三个参数：proxy 是动态代理实例本身，method 是被调用的方法，args 是调用方法时传递的参数。在这个方法的实现中，你可以定义在方法调用前后要执行的逻辑，比如日志记录、权限检查等。你还可以使用 method.invoke(target, args) 来调用目标对象的实际方法。

总的来说，Proxy 类用于创建动态代理实例，而 InvocationHandler 接口用于定义代理实例上方法调用的处理逻辑。通过这两个类的配合使用，可以在运行时动态地创建代理对象并处理其方法调用，而无需手动编写代理类的代码。

​ 但是在Retrofit中使用动态代理获取到api接口中的方法和方法的参数，然后解析注解拼装成一个HttpServiceMethod对象，然后调用这个对象的invoke方法。

​ loadServiceMethod(method)方法解析注解拼装成一个HttpServiceMethod对象，等于替代了原生OkHttp拼装url，参数以及返回值解析等大量的重复代码工作

​ 外部一旦调用了接口方法，例如request.getData();，就会触发InvocationHandler的invoke，从而根据注解生成Retrofit的ServiceMethod和OkHttpCall对象，这些都是发起网络请求的必备信息，然后最终发起请求。

Retrofit大致流程

Retrofit中一次完整的流程包括以下几个步骤：

  1. Retrofit的创建，可以配置OkhttpClient、CallAdapter适配器、Converter转换器等。
  2. 调用create方法动态生成代理类发起网络请求，内部会调用到loadServiceMethod方法获取一个ServiceMethod对象。
  3. 经过ServiceMethod的parseAnnotations来解析请求方法的注解、参数注解、返回类型等数据传递给HttpServiceMethod.
  4. HttpServiceMethod中会根据method、返回类型、注解等获取CallAdapter和Conveter实现类，并返回一个CallAdapted对象。
  5. 回调CallAdapted的invoke方法，获取网络请求的封装类OkHttpCall对象。
  6. OkHttpCall对象会传递给CallAdapter对象来发起网络请求，并在OkHttpCall中调用Conveter解析响应。

Retrofit 实现原理解析

CallAdapter和Conveter的作用

​ 在 Retrofit 中，CallAdapter 和 Converter 是两个用于定制网络请求和响应处理行为的重要组件。它们的实现类分别用于适配不同的调用方式和转换不同的数据格式。

CallAdapter

​ CallAdapter 负责将 Call 类型的响应适配成其他类型，比如 Observable、Future 或自定义类型。这允许你使用不同的异步机制来处理网络请求。

  • 作用：CallAdapter 的作用是将 Retrofit 的 Call 对象转换成其他类型，从而支持不同的异步处理模式。
  • 示例：例如，如果你想在 Retrofit 中使用 RxJava 来处理异步请求，你可以使用 RxJavaCallAdapterFactory 作为 CallAdapter 的实现类。这样，API 接口方法就可以返回 Observable、Single 或其他 RxJava 类型。

Converter

​ Converter 负责请求和响应的数据转换。它将 HTTP 请求体转换为特定格式的数据（比如 JSON、XML），并将响应体转换为 Java 对象。

  • 作用：Converter 的作用是将 Java 对象与请求体或响应体之间进行转换，支持不同的数据格式和序列化库。
  • 示例：例如，如果你的 API 交互使用 JSON 格式，你可以使用 GsonConverterFactory 作为 Converter 的实现类。这样，Retrofit 就会使用 Gson 库来序列化请求参数为 JSON，并将响应 JSON 反序列化为 Java 对象。

​ 总之，CallAdapter 和 Converter 的实现类在 Retrofit 中起着至关重要的作用，它们使得 Retrofit 能够灵活地适配不同的调用方式和数据格式，满足不同场景下的网络请求需求。
