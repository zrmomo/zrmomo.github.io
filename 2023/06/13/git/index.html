更新远程代码到本地：git fetch更新远程仓库最新的版本号到本地的远程仓库然后通过merge合并本地的本地远程的版本号
git fetch –all 获取所有远程仓库所有分支的数据 git merge origin/<远程分支> 因为fetch 后会在本地的版本库中生成每个远程分支的版本号然后合并需要的分支
git fetch origin <远程分支> fetch指定分支
git fetch+git rebase

      git pull 直接将远程仓库同时更新到本地的本地和远程
      git pull origin <远程分支>
      git pull --rebase 以rebase 的方式合并代码
      git pull = git fetch + git merge

提交代码： git push origin dev:dev
冒号前面的dev：推送本地的dev分支到远程origin
冒号后面的dev：远程origin没有会自动创建

      git push -u origin master中-u的用法，加了参数-u后，以后即可直接用git push 代替git push origin master
      git push origin <远程分支>/git push

      更新当前分支代码的时候一定要使用 git pull origin xxx --rebase

git rebabse 和git merge的区别：
git rebase和git merge都是Git用来合并分支代码基本命令
rebase将提交记录合并为一个合入分支
merge将所有记录合入分支
使用 rebase 和 merge 的基本原则：
融合代码到公共分支的时使用git merge,而不用git rebase
融合代码到个人分支的时候使用git rebase，可以不污染分支的提交记录，形成简洁的线性提交历史记录。
更新当前分支的内容时一定要使用 –rebase 参数
例如现有上游分支 master，基于 master 分支拉出来一个开发分支 dev，在 dev 上开发了一段时间后要把 master 分支提交的新内容更新到 dev 分支，此时切换到 dev 分支，使用 git rebase master。 等 dev 分支开发完成了之后，要合并到上游分支 master 上的时候，切换到 master 分支，使用 git merge dev

删除分支 git branch -d <分支名> 删除远程分支 git push origin –delete <分支名>

拉取指定分支的代码 git clone -b <分支名> 地址

回退到前3次提交之前，以此类推，回退到n次提交之前 git reset –hard HEAD~3

git branch log 强制将分支指向log
